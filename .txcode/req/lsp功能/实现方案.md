# LSP 功能实现方案

## 1. 需求概述

### 1.1 功能目标

在 Web 页面提供 LSP 服务器管理功能，允许用户：
- 查看可用的 LSP 服务器列表
- 选择启用/禁用特定的 LSP 服务器
- 默认启用：TypeScript、Python、Java、C++ 四种语言

### 1.2 启动逻辑

1. 用户在 Web 页面启用 LSP 服务器
2. 启动时检查 `~/.txcode/bin/` 是否存在对应服务器
3. **存在则直接启动**
4. **不存在则自动下载，下载完成后再启动**
5. 下载失败时在 Web UI 显示错误状态
6. 重启客户端后，从数据库读取启动状态，自动启动（重复步骤 2-5）

### 1.3 AI 交互

通过 LspTool 暴露给 AI agent：
- `goToDefinition` - 跳转定义
- `findReferences` - 查找引用
- `hover` - 悬停信息
- `documentSymbol` - 文档符号
- `workspaceSymbol` - 工作区符号搜索
- `goToImplementation` - 跳转实现
- `callHierarchy` - 调用层次


### 3.4.1 Java 版本检查实现

```typescript
export async function checkJavaVersion(): Promise<{ valid: boolean; version?: number; error?: string }> {
  try {
    // 1. 检查 java 命令是否存在
    const javaPath = which("java");
    if (!javaPath) {
      return { 
        valid: false, 
        error: "Java 未安装，请安装 Java 21 或更高版本" 
      };
    }
    
    // 2. 获取 Java 版本
    const result = await Process.run(["java", "-version"], { nothrow: true });
    const output = result.stderr.toString() + result.stdout.toString();
    
    // 3. 解析版本号 (e.g., "21.0.2" -> 21)
    const match = output.match(/"(\d+)\.\d+\.\d+"/);
    if (!match) {
      return { 
        valid: false, 
        error: "无法解析 Java 版本" 
      };
    }
    
    const version = parseInt(match[1]);
    
    // 4. 检查版本是否满足要求
    if (version < 21) {
      return { 
        valid: false, 
        version,
        error: `Java 版本过低：${version}，JDTLS 需要 Java 21 或更高版本` 
      };
    }
    
    return { valid: true, version };
  } catch (error) {
    return { 
      valid: false, 
      error: `检查 Java 版本失败: ${error.message}` 
    };
  }
}
```

## 6. 注意事项

1. **完全独立**：不依赖 opencode 的任何代码
2. **JSON-RPC**：使用 vscode-jsonrpc 实现标准 LSP 协议
3. **下载方式多样**：支持 npm、GitHub Releases、go install、gem、dotnet tool 等
4. **进度追踪**：下载过程实时更新进度
5. **错误处理**：单个服务器失败不影响其他服务器
6. **状态持久化**：SQLite 存储配置，重启后生效
7. **Java 版本检查**：JDTLS 需要 Java 21+，启动前必须检查并提示用户
