## 一、文档目录结构

```
docs/
├── 接口文档/           # API 接口文档
│   └── {模块}/
│       ├── 用户列表_api_users_list.md
│       ├── 用户详情_api_users_detail.md
│       └── 保存用户_api_users_save_user.md
├── 测试文档/           # 测试相关文档
│   └── {模块}/
│       └── ...
├── 开发文档/           # 开发指南、架构文档
│   └── ...
└── 规范文档/           # 项目规范文档
    └── ...
```

## 二、文档命名规范

| 类型 | 命名格式 | 示例 |
|------|----------|------|
| 接口文档 | `{接口名称}_{接口路径}.md` | `用户列表_api_users_list.md` |
| 测试文档 | `{模块}_{测试类型}.md` | `user_单元测试.md` |
| 开发文档 | `{主题}_{版本}.md` | `部署指南_v1.0.md` |

**命名规则**：
- 使用中文命名，简洁明确
- 接口路径中的下划线保留
- 版本号使用 `v1.0` 格式

## 三、接口文档规范

### 3.1 文档模板

接口文档必须包含以下部分：
- **基本信息**：请求方法、路径、认证方式
- **请求参数**：Query 参数（GET）或 Body 参数（POST）
- **响应参数**：code、message、data 结构
- **请求示例**
- **响应示例**
- **错误码**

### 3.2 编写要求

- **必填字段必须标注**：参数说明中标注"必填"或"否"
- **类型必须明确**：int、string、array、object、boolean
- **示例必须完整**：包含正常和异常场景
- **错误码必须覆盖**：常见错误都需要说明

### 3.3 示例文件

参考示例：`examples/api-doc-example.md`

## 四、测试文档规范

### 4.1 测试用例格式

测试用例表必须包含：用例编号、用例描述、前置条件、输入、预期输出

### 4.2 测试报告格式

测试报告必须包含：基本信息、测试统计、测试详情、问题记录

### 4.3 示例文件

参考示例：`examples/test-doc-example.md`

## 五、开发文档规范

### 5.1 README 规范

每个项目必须有 README.md，包含：
- 技术栈
- 项目结构
- 快速开始
- API 文档链接
- 配置说明

### 5.2 模块开发文档

模块开发文档包含：功能说明、目录结构、核心接口、业务逻辑

## 六、Markdown 编写规范

### 6.1 格式要求

- 使用标准 Markdown 语法
- 标题层级清晰
- 代码块必须标注语言
- 表格必须对齐

### 6.2 图片规范

- 图片存放在 `docs/images/` 目录
- 使用相对路径引用
- 图片命名：`{模块}_{描述}.png`

## 七、文档维护

- 接口变更必须同步更新文档
- 重大版本更新需要更新 README
- 测试文档在测试完成后编写
